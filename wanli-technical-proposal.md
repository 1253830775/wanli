# AI 万历穿越文字冒险 — 技术方案

## 1. 项目概述

一款由 AI 驱动的历史穿越文字冒险游戏。玩家穿越成明神宗朱翊钧，从隆庆帝托孤（1572 年）开始，凭借现代知识自由决策，改变历史走向。AI 实时生成叙事、角色对白与事件发展，知识图谱确保因果一致性。

## 2. 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue 3 / React 18 |
| 后端 | Java 17+ / Spring Boot 3.x |
| AI | DeepSeek / Kimi 2.6（流式 API） |
| 数据库 | PostgreSQL 15+ + pgvector |
| 实时通信 | SSE (Server-Sent Events) |
| 知识图谱 | PostgreSQL + pgvector（向量化子图检索） |

## 3. 系统架构

![系统架构图](wanli-architecture.png)

### 3.1 展示层（Vue / React）

| 组件 | 职责 |
|------|------|
| 叙事流渲染 | SSE 流式接收并逐 token 渲染 AI 生成的叙事文本 |
| @NPC 交互输入 | 支持 `@NPC名` 唤起特定角色对话，+ 自由文本输入 |
| 世界状态面板 | 实时展示年份、国库（两）、民望、军队忠诚度、当前位置 |
| 因果事件日志 | 展示蝴蝶效应链条（事件A→结果B→影响C） |

### 3.2 业务层（Java / Spring Boot）

| 组件 | 职责 |
|------|------|
| SSE / WebSocket | 维持与前端的长连接，流式推送 AI 响应 |
| 命令解析器 | 解析玩家输入意图（自由行动 / @NPC / 查看状态等） |
| 会话管理器 | 管理世界线状态快照，支持 S/L 回溯 |
| 上下文组装器 | 每轮生成前合并知识图谱子图 + NPC 人格档案 + 世界状态 |

### 3.3 AI 编排层

| 组件 | 职责 |
|------|------|
| LLM 客户端 | 调用 DeepSeek / Kimi 2.6 流式 API |
| Prompt 构建器 | 注入 NPC Persona + 因果链 + 世界状态，构造 System Prompt |
| 知识图谱查询 | 自动检索与当前场景最相关的 Top 50 子图 |
| NPC 人格管理器 | 维护 NPC 人格档案（性格标签、知识边界、态度值演化） |

### 3.4 数据层（PostgreSQL + pgvector）

| 存储 | 内容 |
|------|------|
| 知识图谱 | 实体表（人/地/事件/物品）、关系表（有向边 + 属性）、因果事件表 |
| 世界状态库 | 每个关键节点的结构化 JSON 快照（增量存储） |
| NPC 档案库 | 人格特征、知识范围、对玩家态度（含历史事件影响日志） |
| 会话历史库 | 最近 N 轮对话全文 + 向量化索引（用于 RAG 检索） |

## 4. 核心设计

### 4.1 交互模式（混合型）

```
日常推进：自由输入文本 → AI 生成叙事
关键节点：AI 生成若干选项 → 玩家选择
角色对话：输入 @张居正 → 进入该 NPC 焦点对话模式
场景切换：输入 "去乾清宫" → AI 生成场景过渡叙事
```

### 4.2 知识图谱三层粒度

```
实体层    → 人 / 地 / 事件 / 物品 / 制度
关系层    → (实体A) —[关系]→ (实体B) + 属性
因果事件层 → {事件ID, 触发条件, 影响: [{实体, 新状态}]}
```

每次 AI 生成前，只检索当前上下文最相关的 Top 50 子图。

### 4.3 分层记忆架构

| 层级 | 存储方式 | 用途 |
|------|----------|------|
| 世界状态 | 结构化 JSON | 国库、民望、军事等量化指标 |
| 因果链 | RAG + 向量检索 | 事件A→结果B的因果传播 |
| NPC 人格 | 结构化档案 | 性格、知识边界、态度值 |
| 短期窗口 | 最近 10-15 轮对话全文 | 当前会话连贯性 |

### 4.4 分支爆炸管理

不采用传统分支树，而是**世界状态快照**模式：
- 每个关键决策保存 `(前因, 决策, 新世界状态)` 三元组
- AI 根据当前世界状态实时生成叙事，不存在预计算分支
- S/L 回溯：恢复对应时间点的世界状态快照即可

### 4.5 NPC 人格稳定性

每个关键 NPC 维护结构化档案：

```json
{
  "id": "zhang_juzheng",
  "name": "张居正",
  "traits": ["严厉", "忠诚", "务实", "传统"],
  "knowledge_boundary": ["不知后世之事", "不知玩家穿越身份"],
  "attitude": { "base": 75, "history": [{"event": "玩家支持考成法", "delta": +10}],
  "status": "顾命大臣"
}
```

每轮生成对白时，NPC 档案注入 System Prompt。

### 4.6 延迟优化

- **流式输出**（SSE）：200ms 内完成上下文组装，逐 token 渲染
- **预加载**：玩家阅读当前文本时，后台预查询下一轮图谱
- **缓存策略**：NPC 档案本地向量缓存 + 图谱子图查询缓存（命中率目标 ≥ 70%）
- **增量序列化**：世界状态只保存增量变化

## 5. 交互流程示例

### 玩家自由输入
```
玩家：我要去乾清宫找张居正谈谈考成法的事
系统：正在生成...
AI：你步入乾清宫东暖阁，张居正正伏案批阅奏章。
     他抬头见你进来，起身行礼："陛下驾临，臣有失远迎。"
     [可选：@张居正 "免礼，朕想问问考成法推行得如何了"]
     [可选：@冯保 "去把冯保也叫来"]
     [可选：继续自由输入]
```

### 上朝场景
```
系统：早朝开始。今日议题：黄河水患赈灾方案
     在场大臣：
       @张居正（首辅） @冯保（司礼监）
       @高拱（顾命大臣） @朱衡（工部尚书）
玩家：@张居正 张先生有何建议？
AI：张居正出列，手持奏本："臣以为当疏浚河道，以工代赈..."
```
